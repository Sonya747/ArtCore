
# 一、整体结构

```text
users
workspaces

workflows
generation_tasks

```

# 二、核心设计原则（非常关键）

### 1️⃣ 一切挂在 workspace 下

```text
workspace_id 是核心外键
```

👉 所有资源统一权限控制：

* workflow
* task
* image

---

### 2️⃣ workflow 存 JSON（重点）

直接用 PostgreSQL 的 JSONB：

```sql
workflow_json JSONB
```

👉 这是你系统的核心资产

---

### 3️⃣ task = 调用 ComfyUI 的记录

```text
一次生成 = 一条 task
```

---

# 三、表结构（可直接用）

我用接近生产的 SQL 写法给你。

---

## 1️⃣ users（用户）

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE,
  name TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```


---

---

## 4️⃣ workflows（工作流模板）

```sql
CREATE TABLE workflows (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  workspace_id UUID REFERENCES workspaces(id),

  name TEXT NOT NULL,
  description TEXT,

  -- 核心：ComfyUI API格式 JSON
  workflow_json JSONB NOT NULL,

  -- 可选：定义可修改参数（用于前端表单）
  input_schema JSONB,

  created_by UUID REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

---

## 5️⃣ generation_tasks（生成任务）

👉 这是系统核心表

```sql
CREATE TABLE generation_tasks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  workspace_id UUID REFERENCES workspaces(id),
  workflow_id UUID REFERENCES workflows(id),

  user_id UUID REFERENCES users(id),

  -- 用户输入参数
  input_params JSONB,

  -- 实际执行的 workflow（已注入参数）
  workflow_snapshot JSONB,

  -- ComfyUI 返回
  prompt_id TEXT,

  status TEXT CHECK (status IN (
    'pending',
    'running',
    'success',
    'failed'
  )) DEFAULT 'pending',

  error_message TEXT,

  created_at TIMESTAMP DEFAULT NOW(),
  started_at TIMESTAMP,
  finished_at TIMESTAMP
);
```

---

## 6️⃣ task_outputs（生成结果）

```sql
CREATE TABLE task_outputs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  task_id UUID REFERENCES generation_tasks(id),

  type TEXT, -- image / video / text

  file_url TEXT,     -- OSS / 本地路径
  file_name TEXT,

  metadata JSONB,    -- width / height / seed 等

  created_at TIMESTAMP DEFAULT NOW()
);
```

---

## 7️⃣ assets（用户上传素材，可选）

```sql
CREATE TABLE assets (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  workspace_id UUID REFERENCES workspaces(id),
  user_id UUID REFERENCES users(id),

  file_url TEXT,
  file_type TEXT, -- image / mask / reference

  metadata JSONB,

  created_at TIMESTAMP DEFAULT NOW()
);
```

# 六、关键字段设计说明（非常重要）

---

## 1️⃣ workflow_json vs workflow_snapshot

| 字段                | 作用     |
| ----------------- | ------ |
| workflow_json     | 模板     |
| workflow_snapshot | 实际执行版本 |

👉 为什么要 snapshot？

因为：

> 用户执行后 workflow 可能会变

否则历史记录会错乱。

---

## 2️⃣ input_params

```json
{
  "prompt": "a girl",
  "width": 1024
}
```

👉 用于：

* UI 展示
* 复现任务

---

## 3️⃣ status 设计

```text
pending → running → success / failed
```

👉 后续可以扩展：

```text
queued / canceled
```
